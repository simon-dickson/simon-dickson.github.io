Fundamental Terms & Definitions

  • Interpretation vs. Compilation
  • Interpreter vs. Compiler
  • Lexis, Syntax, Semantics
concept.py
# Python is INTERPRETED — runs line by line
# No compile step needed; .py → CPython bytecode

# LEXIS  = valid characters & tokens
# SYNTAX = grammar rules (structure)
# SEMANTICS = meaning of the code

# SyntaxError — wrong structure:
# print "hello"   ← valid Python 2, error in 3

# SemanticError — wrong meaning:
x = 1 / 0  # ZeroDivisionError at runtime
Key fact: Python uses an interpreter. Source code is compiled to bytecode first, then the interpreter (CPython) executes that bytecode.

PCEP 1.4

Operators & Data Types

  • Arithmetic: ** * / % // + –

  • Unary / binary, priorities, binding
  • Bitwise: ~ & ^ | << >>

operators.py
# Arithmetic — KNOW the precedence!
print(2 ** 3)     # 8  (highest priority)
print(7 // 2)     # 3  (floor division)
print(-7 // 2)    # -4 (rounds toward -∞)
print(7 % 2)      # 1  (modulo)

# Bitwise operators
print(5 & 3)      # 1   (AND: 101 & 011 = 001)
print(5 | 3)      # 7   (OR:  101 | 011 = 111)
print(5 ^ 3)      # 6   (XOR: 101 ^ 011 = 110)
print(~5)         # -6  (NOT: -(n+1))
print(5 << 1)     # 10  (left shift = *2)
print(5 >> 1)     # 2   (right shift = //2)

# Short-circuit evaluation
x = None
result = x or "default"   # → "default"
check  = x and x.value    # safe — no AttributeError

PCEP 2.2

while Loop

  • while condition, while-else
  • break, continue, pass
  • Infinite loops with break
while_loop.py
i = 0
while i < 5:
    print(i, end=" ")    # 0 1 2 3 4
    i += 1
else:
    print("\ndone")    # runs if no break

# break exits the loop (else doesn't run)
while True:
    user = input("Type 'quit': ")
    if user == "quit":
        break

# continue skips to next iteration
i = 0
while i < 5:
    i += 1
    if i == 3:
        continue          # skip 3
    print(i, end=" ")  # 1 2 4 5
PCEP 2.3

for Loop & range()

  • for-in, for-else
  • range(start, stop, step)
  • Nested loops
  • break and continue in for loops
for_loop.py
# range() — stop is EXCLUSIVE
for i in range(5):
    print(i, end=" ")      # 0 1 2 3 4

for i in range(1, 6):
    print(i, end=" ")      # 1 2 3 4 5

for i in range(10, 0, -2):
    print(i, end=" ")      # 10 8 6 4 2

# for-else: else runs if no break
for n in [2, 3, 5]:
    if n == 4:
        break
else:
    print("4 not found")   # ← runs

# Nested loops — multiplication table
for r in range(1, 4):
    for c in range(1, 4):
        print(f"{r*c:3}", end="")
    print()
Remember: range(5) = 0,1,2,3,4. The stop value is never included.

🔄 Loop Control — Quick Reference
break
Exit loop immediately. The else clause does NOT run.
continue
Skip rest of loop body, jump to next iteration.
pass
Do nothing — placeholder for empty blocks.
while-else
else runs only if loop ends without break.
for-else
else runs only if loop ends without break.
range(n)
0 to n-1. range(a,b,s): start, stop (exclusive), step.

Defining & Invoking Functions

  • def keyword, return statement
  • None — returned when no return
  • Recursion
functions.py
def greet(name):
    return f"Hello, {name}!"

print(greet("Alice"))   # Hello, Alice!

# Function with no return → returns None
def say_hi():
    print("Hi")

result = say_hi()
print(result)      # None

# Multiple return values → returns a tuple
def min_max(lst):
    return min(lst), max(lst)

lo, hi = min_max([3, 1, 4, 1, 5])

# Recursion — factorial
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

print(factorial(5))   # 120
PCEP 4.2

Parameters, Arguments & Scope

  • Positional, keyword, mixed arguments
  • Default parameter values
  • Name scope — local vs global
  • global keyword
  • Name shadowing
scope.py
# Default parameters
def power(base, exp=2):
    return base ** exp

print(power(3))       # 9  (exp defaults to 2)
print(power(2, 10))  # 1024

# Keyword arguments
power(exp=3, base=4)   # order doesn't matter

# SCOPE — LEGB rule
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"   # shadows outer x
        print(x)      # "local"
    inner()
    print(x)          # "enclosing"

outer()
print(x)              # "global"

# global keyword
counter = 0
def increment():
    global counter
    counter += 1    # modifies global
LEGB Rule: Python looks up names in: Local → Enclosing → Global → Built-in

PCEP 4.4

Exception Handling: try / except

  • try-except block
  • Multiple except clauses — order matters
  • else clause (runs if no exception)
  • finally clause (always runs)
  • Propagating exceptions
  • raise statement
exception_handling.py
try:
    x = int(input("Enter a number: "))
    result = 100 / x
except ValueError:
    print("Not a valid number!")
except ZeroDivisionError:
    print("Cannot divide by zero!")
except Exception as e:
    print(f"Unexpected: {e}")
else:
    print(f"Result: {result}")   # no exception
finally:
    print("Always runs")          # cleanup

# raise — re-raise or raise custom
def validate(n):
    if n < 0:
        raise ValueError("Must be positive")
    return n

# Propagation — if no except matches,
# exception bubbles up to caller
def outer():
    try:
        inner()           # inner raises
    except ValueError:
        print("caught here")

def inner():
    raise ValueError("from inner")
Order matters! Put specific exceptions BEFORE general ones. ZeroDivisionError before Exception, not after — or it will never be reached.
🔴 Exception Hierarchy (PCEP relevant)
BaseException
Root of all exceptions. Rarely caught directly.
SystemExit
Raised by sys.exit(). Subclass of BaseException.
KeyboardInterrupt
Ctrl+C. Subclass of BaseException.
Exception
Base for all "normal" exceptions. catch-all.
ArithmeticError
Parent of ZeroDivisionError, OverflowError.
LookupError
Parent of IndexError and KeyError.
TypeError
Wrong type: "a" + 1 → TypeError
ValueError
Right type, wrong value: int("abc")
AssertionError
Raised by a failing assert statement.
NameError
Variable not defined.

⚠️ Common Exam Traps
sort() vs sorted()
sort() returns None and modifies in-place. sorted() returns a new list.
Floor division with negatives
-7 // 2 = -4 (rounds toward -infinity, not zero)
Boolean is a subclass of int
True == 1, False == 0. True + True = 2.
Mutable default arguments
def f(lst=[]): bad! The same list is shared across calls.
range stop is exclusive
range(1, 5) = 1, 2, 3, 4 — never includes 5.
Single-element tuple
(42,) is a tuple. (42) is just parentheses around an int.
else on loops
Loop else only skipped if loop exits via break.
is vs ==
is checks identity (same object). == checks value equality.

Practice Quiz

🧠 Test Your Knowledge — Click an option to check

What is the output of:

print(7 // -2)
B) -4 — Floor division rounds toward negative infinity. 7/-2 = -3.5, rounded down = -4.
Which of the following is a valid one-element tuple?
B) (42,) — The trailing comma is required. (42) is just an integer in parentheses.
What does this print?

x = [1, 2, 3]
y = x
y.append(4)
print(x)
B) [1, 2, 3, 4] — y = x assigns the same reference. Both x and y point to the same list object in memory.
What value does input() always return?
C) str — input() always returns a string, regardless of what the user enters. You must convert with int() or float().
What is the output of:

for i in range(3):
    pass
else:
    print("done")
B) done — The loop's else clause runs when the loop completes without a break statement.
Which exception is raised by: int("hello")
B) ValueError — TypeError is for wrong type operations. "hello" is the right type (str) but not a valid integer value, so ValueError.
What is the output of:

print(2 ** 3 ** 2)
B) 512 — ** is right-associative. 3**2 = 9 first, then 2**9 = 512. NOT (2**3)**2 = 64.
What does list.sort() return?
A) None — sort() modifies the list in-place and returns None. Use sorted(lst) if you need a new sorted list returned.
